# 14교시 — 정리: 전체 되짚기 + 배포·확장·다음 단계

> **이 교시가 끝나면** 2일 동안 손으로 쌓고 AI로 가속해 만든 보이스 에이전트의 **전체 구조가 한눈에** 정리되고, 이걸 어디에·어떻게 **배포·확장**할지, 그리고 다음에 무엇을 배우면 좋을지에 대한 지도를 갖게 됩니다.

---

## 0. 이 교시의 성격

이 교시는 새 코드를 쓰지 않습니다. **되짚고, 잇고, 내다봅니다.** 그동안 만든 조각들이 어떻게 하나의 시스템이 되었는지 확인하고, 실제 사용·배포로 넘어가는 다리와, 더 나아갈 방향을 정리합니다.

---

## 1. 우리가 만든 것 (한 문장)

**브라우저 마이크로 말을 걸면, 서버가 OpenAI Realtime과 실시간으로 음성을 주고받고, 필요하면 도구(시간·날씨·계산·환율·온도·길이)를 안전하게 호출해 답하며, 대화를 저장하는 — Clean Architecture와 TDD로 지은 음성 에이전트.**

## 2. 전체 구조 한눈에

### 계층 (Clean Architecture)

```
┌─────────────────────────────────────────────────────────┐
│ infra    voice_handler(FastRTC 핸들러), voice_app        │  ← 바깥: IO·결선
│          큐·재생·바지인·이벤트 루프·저장 결선            │
├─────────────────────────────────────────────────────────┤
│ adapter  Realtime 세션·연결, 도구·안전실행·변환,         │  ← 통신·형식·구현
│          가드레일 파이프라인, 대화 저장(파일·마크다운)   │
├─────────────────────────────────────────────────────────┤
│ usecase  SendUserUtterance, StartConversation,           │  ← 흐름·규칙 조립
│          ListActiveTools, SaveConversation + 포트(약속)  │
├─────────────────────────────────────────────────────────┤
│ domain   Turn·Conversation·AudioFrame·SessionConfig,     │  ← 심장: 순수 규칙
│          ToolSpec·ToolToggleState·RateLimiter·요약·검증  │
└─────────────────────────────────────────────────────────┘
     의존 방향: 바깥 → 안. 안쪽(domain)은 바깥을 절대 모른다.
```

### 흐름 (음성 한 턴 + 도구)

```
[마이크] → FastRTC.receive → AsyncRealtimeSession.send_audio(PCM→base64)
   → OpenAI Realtime (서버 VAD가 말끝 감지)
   ← 이벤트 스트림
        ├─ 자막 delta → 화면/버퍼
        ├─ 음성 delta → 재생 큐 → emit → 스피커
        ├─ speech_started → 바지인(큐 비우기 + 조건부 취소)
        └─ function_call → 🛡️ dispatch_tool(빈도→존재→인자→실행) → 결과 반환 → response.create
   → (세션 종료) SaveConversation → transcripts/*.md + 요약
```

### 교시별 산출물

| 교시 | 핵심 산출물 | 계층 |
|---|---|---|
| 1 | 개발환경·오디오 스모크 | infra |
| 2 | Turn·Conversation·AudioFrame·SessionConfig·ToolSpec·ToolToggleState | domain |
| 3 | 유즈케이스·포트·Fake·ListActiveTools | usecase |
| 4 | OpenAI Realtime 어댑터(텍스트) | adapter |
| 5 | async 세션·FastRTC 오디오 입력·자막 | adapter·infra |
| 6 | 음성 재생·바지인 = MVP | infra |
| 7 | 커스텀 웹 UI 셸 | infra(프론트) |
| 8 | 도구(함수 호출) + 토글 #1 | adapter·infra |
| 9 | 계산기 + 가드레일(검증·안전실행) | adapter·domain |
| 10 | 환율 + 정책 가드레일(허용목록·빈도제한) | adapter·domain |
| 11 | 온도 + 가드레일 파이프라인 | adapter |
| 12 | 바이브 코딩 ①(길이 도구) | 전 계층 |
| 13 | 바이브 코딩 ②(대화 저장·요약) | 전 계층 |

## 3. 관통하는 원칙 다섯 가지

**① 안쪽은 바깥을 모른다.** 도메인은 순수 규칙만. 통신·파일·프레임워크는 바깥 계층에. 그래서 OpenAI가 바뀌어도, 파일→클라우드로 바꿔도 심장은 안 흔들립니다.

**② 약속(포트)에 기대고, 구현은 갈아 끼운다.** 유즈케이스는 포트에만 의존. 진짜/가짜를 자유롭게 교체해, 테스트는 인터넷·비용 없이 돌았습니다.

**③ 테스트 먼저.** 실패하는 테스트(RED)로 정답을 못박고 최소 구현(GREEN). 실시간 결선은 "실행·관찰"로 나눠, 검증 가능한 건 전부 테스트로 덮었습니다.

**④ 바깥 입력은 믿지 않는다.** 사용자·모델·AI가 준 값은 신뢰 경계 밖. 정규화 → 허용 목록 → 범위 → 빈도를 한 파이프라인으로 통과시켜 안전하게 실행했습니다.

**⑤ AI는 가속기, 방향키는 사람.** 견고한 토대 위에서만 바이브 코딩이 안전합니다. 규칙·스펙·테스트를 먼저 주고, 리뷰로 승인했습니다.

## 4. 자가 점검 (전체 게이트)

배포·공유 전에 아래를 확인합니다.

- [ ] `uv run pytest` — 전체 초록(대략 80여 개 이상)
- [ ] `uv run ruff check src/voice_agent` — All checks passed!
- [ ] `/`(커스텀 UI) 또는 `/ui`에서 음성 대화가 되고, 음성이 재생되며, 끼어들면 멈춘다
- [ ] "지금 몇 시/서울 날씨/3 곱하기 7/100달러 원화/섭씨 100도/3km" 각 도구가 호출된다
- [ ] 허용 밖 입력·누락 인자·반복 호출이 가드레일에 막힌다
- [ ] 세션 종료 시 `transcripts/*.md`가 생기고 요약이 뜬다
- [ ] 계층 규칙 위반이 없다(도메인에 IO 없음, 유즈케이스에 어댑터 import 없음)

## 5. 배포로 가는 다리

교육용 로컬 실행(`uvicorn ... 127.0.0.1`)에서 실제 서비스로 넘어갈 때의 **점검 항목**입니다(개념 안내이며, 구체 절차는 환경마다 다릅니다).

**① 비밀 관리.** `OPENAI_API_KEY`를 코드·저장소에 넣지 않기. `.env`는 로컬용, 배포는 플랫폼의 비밀 저장소(환경변수)로. 비용 상한·사용량 알림을 걸어두기.

**② HTTPS·마이크 권한.** 브라우저는 보안(HTTPS) 컨텍스트에서만 마이크를 허용합니다. 배포 시 TLS가 필요합니다.

**③ 다중 접속.** `copy()`가 접속자마다 새 핸들러를 주지만, 서버 자원(동시 세션 수)·비용을 고려해 한도를 둡니다. 빈도 제한을 세션·사용자 단위로 확장하는 것도 검토합니다.

**④ 관측(observability).** 지금은 터미널 로그입니다. 실서비스라면 구조화 로그·오류 추적·지표(응답 지연·도구 호출 수·실패율)를 붙입니다.

**⑤ 실패 처리.** 네트워크 끊김·모델 오류·저장 실패에 대한 폴백과 사용자 안내. 이미 만든 "종료는 정상", "저장 실패 격리"가 그 출발점입니다.

**⑥ 데이터·프라이버시.** 대화 저장은 개인정보일 수 있습니다. 보관 기간·접근 권한·삭제 정책을 정합니다.

> ⚠️ 이 항목들은 방향 안내입니다. 배포 대상(사내 서버·클라우드·컨테이너)에 따라 구체 방법이 달라지므로, 대상이 정해지면 그에 맞춰 별도 점검이 필요합니다.

## 6. 확장 아이디어 (다음에 붙일 것)

지금 구조라면 아래는 **패턴 재사용**으로 붙습니다.

- **도구 추가**: 함수 + 명세 + 토글 + 테스트. 가드레일은 파이프라인이 자동 적용(8~11교시).
- **실제 외부 API**: `get_weather`·환율을 진짜 API로 교체(어댑터만 수정, 유즈케이스 불변).
- **웹 자막 표시**: 서버 자막을 SSE로 브라우저에 흘려 화면에 표시(7교시 선택미션 확장).
- **대화 이어가기**: 저장한 대화를 불러와 다음 세션의 맥락으로(유즈케이스 추가).
- **모델 요약**: 저장 시 모델에게 한 문장 요약을 요청(13교시 선택미션).
- **권한/역할**: 사용자별로 켤 수 있는 도구를 다르게(토글 정책 확장).

## 7. 회고 질문 (팀/개인)

- 어느 계층이 가장 자주 바뀌었나? 그 변화가 다른 계층으로 **번지지 않았나?** (경계가 잘 섰다는 신호)
- 어떤 버그가 "실행·관찰" 단계에서만 드러났나? 그걸 다음엔 **테스트로 앞당길** 수 있나?
- 바이브 코딩에서 리뷰가 **실제로 잡아낸** 문제는? 규칙 파일에 무엇을 더 넣으면 좋을까?
- 가드레일 중 실제로 발동한 것은? 빠진 위험 시나리오는 없나?

## 8. 더 배우면 좋은 것

- **스펙 주도 개발(SDD)**: 스펙·헌장(constitution)으로 AI 협업을 체계화.
- **에이전트 하니스**: 도구·메모리·플래닝을 갖춘 자율 에이전트 설계.
- **평가(eval)**: 음성 응답 품질·도구 정확도를 자동으로 측정.
- **관측·비용 최적화**: 실시간 음성의 지연·비용을 다루는 운영 기법.
- **보안 심화**: 프롬프트 주입·데이터 유출 방어를 에이전트 맥락에서.

## 📖 이 과정에서 익힌 핵심 용어 (총정리)

- **Clean Architecture / 계층**: domain·usecase·adapter·infra로 관심사를 나누는 설계.
- **포트/어댑터**: 약속(인터페이스)과 그 구현. 갈아 끼우기 가능.
- **TDD / Test-First**: 실패하는 테스트로 정답을 먼저 고정하는 개발.
- **async / 이벤트 루프 / 큐**: 여러 일을 동시에 흐르게 하는 비동기와 재생 큐.
- **WebRTC / SDP / ICE / 데이터 채널**: 브라우저 실시간 연결의 요소.
- **Realtime 이벤트 / 함수 호출**: session.update·delta·function_call_output·response.create.
- **가드레일**: 검증·안전 실행·허용 목록·빈도 제한을 파이프라인으로.
- **바이브 코딩**: 규칙·스펙·테스트·리뷰로 통제하는 AI 페어 코딩.
